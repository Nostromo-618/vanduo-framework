Changelog

All notable changes to Vanduo Framework are documented here.

v1.1.1 February 8, 2026

Bug Fixes

  • Theme switcher preference guard

    Prevented theme customizer sync from writing to an uninitialized theme switcher state during component startup order.

  • Safe storage fallbacks

    Theme and font preferences now use guarded storage access to avoid crashes when localStorage is unavailable.

  • Pagination class consistency

    Rendered items now include vd- prefixed classes so click handling and styling stay aligned.

Improvements

  • Manual initialization is consistent

    Removed component auto-init blocks so Vanduo.init() is the single entry point.

  • Idempotent init and cleanup hooks

    Repeated Vanduo.init() calls no longer duplicate listeners, and cleanup is standardized across components.

  • DestroyAll coverage expanded

    Search and Image Box now participate in Vanduo.destroyAll() for SPA cleanup.

v1.1.0 February 7, 2026 Breaking Changes

Breaking Changes

  • CSS Class Prefixing

    All CSS classes now use the vd- prefix to prevent collisions with other frameworks. Examples: .btn → .vd-btn, .container → .vd-container, .card → .vd-card. See MIGRATION.md for complete class mapping.

  • Manual Initialization Required

    Framework no longer auto-initializes on page load. You must explicitly call Vanduo.init() after loading the script. This allows better control over initialization timing in SPAs and complex applications.

New Features

  • Multiple Module Formats

    Added ESM (vanduo.esm.js) and CJS (vanduo.cjs.js) build outputs alongside IIFE format. Modern bundlers can now tree-shake unused components.

  • Package.json Exports Map

    Added proper exports field in package.json for clean imports: import { Vanduo } from 'vanduo-framework'.

Improvements

  • CSS Namespace Isolation

    The vd- prefix ensures Vanduo can be safely embedded in applications using Bootstrap, Tailwind, or other CSS frameworks without class name collisions.

v1.0.4 February 7, 2026 Latest

Improvements

  • Image Box Dynamic Sizing

    Enlarged images now display with a maximum size of 50% of the viewport (50vw/50vh) instead of filling the entire screen. Mobile breakpoints use 90vw/80vh for better usability on small screens.

  • Broken Image Styling

    Added graceful fallback styling for broken or missing images. Broken images now display with a subtle gradient background, dashed border, and centered image icon instead of the browser's default broken image icon.

Documentation

  • Image Box Placeholder Images

    Added 9 placeholder images for documentation code examples, eliminating 404 console errors. Placeholders feature dark theme with teal accents matching the framework aesthetic.

  • Updated CSS Customization Docs

    Documentation now reflects the new default values for --image-box-max-width (50vw) and --image-box-max-height (50vh).

  • Documentation Page UX Improvements

    Added Phosphor icons to all 26 section headers for visual consistency with search results. Improved section spacing with uniform divider lines and primary-colored header underlines. Added code snippet examples to Collections, Badges, Chips, Avatars, and Skeleton sections.

v1.0.3 February 1, 2026

New Features

  • Google Fonts Collection

    Added 4 new self-hosted Google Fonts: Ubuntu, Open Sans, Rubik, and Titillium Web. All fonts are privacy-friendly with no external requests. Total font collection now includes 10 fonts.

  • Card Glow Effect

    Added gentle glow effect to cards on hover with .vd-card-glow class. The glow color dynamically adapts to the theme's primary color via Theme Customizer. Includes reusable glow utilities (.vd-hover-glow, .vd-glow-success, etc.) for any element.

Improvements

  • Theme Customizer Font Options

    Updated Theme Customizer with all 10 font options in the font family dropdown.

  • Font Switcher Updates

    Extended Font Switcher component to support the new Google Fonts collection.

Documentation

  • Typography Documentation

    Added comprehensive font families section to documentation with descriptions of all 10 available fonts.

v1.0.2 February 1, 2026

New Features

  • Search Component

    New client-side search with keyboard navigation (⌘K/Ctrl+K), debounced input, category grouping, and full ARIA accessibility. Includes Search.create() factory for custom instances.

Improvements

  • Search component refactored

    Refactored to be more generic and configurable with custom data sources, callbacks, and category icons.

  • ReDoS protection

    Added protection against Regular Expression Denial of Service in search highlight function.

  • Enhanced ARIA attributes

    Added aria-controls, aria-expanded, and role="listbox" to search component for better screen reader support.

Bug Fixes

  • Removed console.log from production

    Removed debug console.log statement from doc-search.js.

Documentation

  • Search component documentation

    Added comprehensive documentation for the Search component including quick start, configuration options, JavaScript API, and CSS customization.

v1.0.1 January 31, 2026

Bug Fixes

  • Vertical button group alignment

    Fixed .vd-btn-group-vertical alignment by changing align-items to stretch for consistent button widths.

  • Dark mode table styling

    Tables now have proper dark mode backgrounds. Added [data-theme="dark"] overrides for table variables.

  • Range slider track visibility

    Range slider track is now visible in dark mode using --color-gray-400 for better contrast.

Layout Changes

  • Getting Started cards layout

    Changed to full-width stacked cards for better readability.

Documentation

  • Separate changelog page

    Created a dedicated changelog page with beautiful styling and icons.

v1.0.0 January 30, 2026 Initial Release

Features

  • Complete UI Component Library

    Buttons, cards, forms, navigation, modals, tabs, toasts, and more.

  • Dark Mode Support

    Built-in dark theme with automatic system preference detection.

  • Theme Customizer

    Real-time customization of colors, fonts, and spacing.

  • Responsive Grid System

    12-column grid with flexible breakpoints.

  • Golden Ratio Typography

    Mathematically harmonious type scale based on Fibonacci sequence.

  • Zero Dependencies

    Pure HTML, CSS, and JavaScript with no external dependencies.

  • Phosphor Icons

    1,500+ icons in 6 weights bundled locally.

  • Accessibility First

    ARIA attributes, focus states, and semantic HTML built-in.
