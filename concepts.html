Our Philosophy

Vanduo is built on a foundation of simplicity, transparency, and developer empowerment. We believe that a CSS framework should enhance your workflow without adding unnecessary complexity or hidden dependencies.

Pure & Simple

Just HTML, CSS, and vanilla JavaScript. No build tools required, no transpilation needed.

Conflict-Free

The vd- prefix ensures our classes never conflict with your existing styles or other libraries.

Modular

Use only what you need. Import individual components or the full framework—your choice.

Developer First

Clean, readable code with consistent patterns. Easy to customize and extend.

What "Vanduo" Means

Vanduo (pronounced van-doo-oh) is derived from Lithuanian: vanduo means "water."

Just as water is essential for life, we believe a good CSS framework should be essential yet unobtrusive—flowing naturally into your projects without forcing you to change your workflow. Water adapts to its container; Vanduo adapts to your design.

CSS Namespacing Strategy

One of the most important architectural decisions in Vanduo v1.1.0 is the introduction of comprehensive CSS namespacing. Every framework class is prefixed with vd- to prevent collisions and ensure safe integration.

Before v1.1.0

/* Generic class names */
.container { }
.row { }
.col-6 { }
.btn { }
.card { }
.alert { }

/* Problems:
   - Conflicts with Bootstrap
   - Conflicts with Tailwind
   - Conflicts with your own .btn
   - Unpredictable in large apps
*/

Since v1.1.0

/* Namespaced class names */
.vd-container { }
.vd-row { }
.vd-col-6 { }
.vd-btn { }
.vd-card { }
.vd-alert { }

/* Benefits:
   - Zero conflicts guaranteed
   - Clear framework origin
   - Safe embedding in any app
   - Multiple framework friendly
*/

Why This Matters

Note: the "Before v1.1.0" snippet above uses legacy, unprefixed class names for comparison only.

Without Namespacing

  • Cannot safely embed in WordPress themes
  • Conflicts with Bootstrap-based admin panels
  • Cannot use alongside Tailwind projects
  • Your .btn styles may override ours
  • Difficult to isolate in micro-frontends

With vd- Prefix

  • Safe embedding in any environment
  • Works alongside Bootstrap, Tailwind, etc.
  • Your .btn remains untouched
  • Shadow DOM friendly
  • Perfect for widget/embed scenarios

Design Tokens

Design tokens are the single source of truth for our visual design system. All CSS custom properties (variables) are prefixed with --vd- for clarity and consistency.

Token Categories

Category Prefix Example
Colors --vd-color- --vd-color-primary
Backgrounds --vd-bg- --vd-bg-primary
Text --vd-text- --vd-text-primary
Typography --vd-font- --vd-font-family-base
Spacing --vd-spacing- --vd-spacing-8
Borders --vd-border- --vd-border-color
Shadows --vd-shadow- --vd-shadow-md
Breakpoints --vd-breakpoint- --vd-breakpoint-md

Using Tokens

/* Override primary color */
:root {
  --vd-color-primary: #6366f1;
  --vd-color-primary-dark: #4f46e5;
}

/* Use in your custom CSS */
.my-component {
  background: var(--vd-color-primary);
  padding: var(--vd-spacing-8);
  border-radius: var(--vd-btn-border-radius);
  font-family: var(--vd-font-family-base);
}

Lifecycle Management

Vanduo provides a lightweight lifecycle manager that enables proper cleanup of event listeners, observers, and other resources. This is essential for Single Page Applications (SPAs) where components are dynamically mounted and unmounted.

Manual Initialization

Unlike earlier versions, Vanduo v1.1.0 requires manual initialization. This gives you full control over when components are initialized and destroyed.

<!-- Include Vanduo -->
<link rel="stylesheet" href="css/vanduo.css">
<script src="js/vanduo.js"></script>
<script>
    // Initialize when ready
    Vanduo.init();

    // Later, when cleaning up (e.g., route change)
    Vanduo.destroyAll();
</script>

Lifecycle Methods

Method Purpose
Vanduo.init() Initialize all registered components
Vanduo.destroyAll() Clean up all components
Vanduo.reinit(name) Reinitialize a specific component
Vanduo.register(name, cfg) Register a custom component

SPA Example

// Router navigation handler
function onRouteChange(newRoute) {
  // Clean up previous page
  Vanduo.destroyAll();
  
  // Load new content
  loadContent(newRoute);
  
  // Initialize Vanduo on new DOM
  Vanduo.init();
}

// Cleanup on page unload
window.addEventListener('beforeunload', () => {
  Vanduo.destroyAll();
});
Note: The lifecycle manager is approximately 100 lines of code and uses a simple Map-based registry. This is not a framework—just a lightweight utility for resource cleanup.
Tip: When adding new DOM in SPAs, run Vanduo.destroyAll() (or a component's destroyAll()) before Vanduo.reinit(name) so cached instances are rebuilt for new elements.

Module Formats

Vanduo v1.1.0 supports three module formats to work with any project setup, from simple static sites to modern bundlers.

IIFE

For traditional script tags. Creates a global Vanduo object. 

<script src="dist/vanduo.min.js"></script>
<script>
    Vanduo.init();
</script>

ESM

For modern bundlers and native ES modules. Enables tree-shaking.

<script type="module">
    import { Vanduo } from './dist/vanduo.esm.js';
    Vanduo.init();
</script>

CJS

For Node.js environments and older bundlers.

const { Vanduo } = require('./dist/vanduo.cjs.js');

// Initialize
Vanduo.init();

Package Exports

// package.json
{
  "exports": {
    ".": {
      "import": "./dist/vanduo.esm.js",
      "require": "./dist/vanduo.cjs.js"
    },
    "./css": "./dist/vanduo.min.css",
    "./css/components/*": "./css/components/*",
    "./js": {
      "import": "./dist/vanduo.esm.js",
      "require": "./dist/vanduo.cjs.js"
    }
  }
}

Migration Guide

Upgrading from v1.0.x to v1.1.0 requires updating class names and initialization code. This guide will help you migrate smoothly.

Quick Migration Script

We've created a Node.js migration script to help automate the class name updates:

# Download the migration script
curl -O https://vanduo.dev/migrate-v1.1.0.js

# Run on your project
node migrate-v1.1.0.js ./my-project

# Review changes
git diff

The script handles:

  • Adding vd- prefix to all framework classes
  • Updating CSS variable references
  • Converting initialization code

Breaking Changes

Summary of Breaking Changes in v1.1.0

Change v1.0.x v1.1.0 Impact
CSS Classes .btn, .card .vd-btn, .vd-card High — All HTML needs updating
CSS Variables --color-primary --vd-color-primary Medium — Custom CSS needs updating
Initialization Auto-init on page load Manual Vanduo.init() High — JS code needs updating
Module Format IIFE only IIFE, ESM, CJS Low — Backward compatible
Lifecycle No cleanup API Vanduo.destroyAll() Low — New feature
